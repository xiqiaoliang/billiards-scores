# 手机端台球追分记分系统 需求文档

## 一、文档概述

### 1.1 项目背景

本系统为移动端竖屏专用双人台球追分记分工具，适配线下实战计分习惯。系统整合计分规则、数据统计、对局管理、时间记录能力，采用IndexedDB实现本地数据持久化，支持多轮整场对局管理、历史记录留存回看，同时预留服务端数据上报扩展能力。整体字段精简、规则标准化，搭配双重校验机制，保证计分准确、操作便捷。

### 1.2 适用人群

双人台球追分对战选手、现场记分人员、台球休闲爱好者。

### 1.3 核心定位

固定双人模式移动端专业记分工具，集计分、规则校验、单局时间统计、整场对局管理、本地数据存储于一体，支持新建对局、封存比赛、历史记录查看，离线可用且数据不丢失，可后续扩展服务端数据同步功能。

## 二、整体功能需求

### 2.1 对战模式

页面打开直接进入**固定双人对战界面**，无模式、人数切换功能，全程仅支持两名选手对战记分。

### 2.2 选手配色规范

配色全局统一，应用于选手名称、计分标签、统计表格、历史明细等全页面场景：

- 选手1：蓝色 `#1677FF`
- 选手2：红色 `#F53F3F`

### 2.3 计分项目与基础分值

所有计分项分值固定，犯规类为扣分项，其余为得分项：

1. 犯规：-1分/次，选择次数无上限
2. 开球犯规：-1分/次，单局每名选手最多选择1次
3. 分球：+2分/局，单局每名选手最多选择1次
4. 普胜：+4分/局
5. 小金：+7分/局
6. 大金：+10分/局

**补充说明**：总览页面合并统计犯规、开球犯规总次数；仅逐局历史明细区分两种犯规类型。

### 2.4 计分标签选择规则

常规模式、让杆模式通用以下规则，系统对违规操作做前置拦截，禁止违规标签添加：

1. 犯规：可多次重复选择，无次数限制。
2. 开球犯规、分球：单局单选手仅可选择1次，可选择不添加。
3. 普胜、小金、大金：三项完全互斥，单局单选手仅能选择其中一项，也可全部不选。
4. 让杆对应项目（让杆分球、让杆普胜、让杆小金、让杆大金），完全沿用上述次数、互斥规则。

### 2.5 让杆加倍与分数统计规则

1. **常规得分**：按基础分值计算，仅累加至基础统计列，不计入额外分。
2. **让杆得分**：实际得分 = 基础分值 × 2
  - 基础统计列：累加原始基础分值
  - 额外分列：累加加倍差值（实际得分 - 基础分值）
3. **总分**：选手总分 = 历史及本局所有常规得分 + 所有让杆实际得分（含扣分）
4. **额外分**：选手额外分 = 全部让杆项目产生的加倍差值总和
5. **净分**：净分 = 选手1总分 − 选手2总分；正数代表选手1领先，负数代表选手2领先，全局唯一展示。

### 2.6 双重提交校验规则

系统采用**标签点击前置拦截 + 提交按钮二次全量校验**机制，校验失败展示对应提示并终止操作，校验通过方可完成提交：

1. 同一选手单局开球犯规选择超过1次：提示「本局开球犯规仅可选择1次」。
2. 同一选手单局分球选择超过1次：提示「本局分球仅可选择1次」。
3. 同一选手单局普胜、小金、大金（含对应让杆项）选择多项：提示「胜负类选项只能选择一项」。
4. 本局待提交区域无任何计分标签时，提交按钮置灰，点击无响应。

### 2.7 单局时间与时长规则

每一局自动记录时间信息，数据随对局记录永久保存，**时间不支持手动编辑**：

1. 本局开始时间：整场比赛第一局，开始时间为大场创建时间；后续对局，开始时间 = 上一局结束时间。
2. 本局结束时间：点击【提交本局成绩】并校验通过的系统当前时间。
3. 对局时长：由结束时间减去开始时间计算得出，展示格式为 时:分:秒。

### 2.8 对局层级管理

系统分为**大场对局（整场比赛）**和**小局（单局对局）**两级结构：

1. 一个大场对局包含多轮单局、全场汇总数据、对局状态、创建时间等信息。
2. 所有单局数据归属对应大场，关联局号、计分明细、时间、时长。
3. 对局状态分为「进行中」「已封存」两种。

### 2.9 本地数据存储规则

1. 存储方案：使用前端IndexedDB实现离线本地存储，页面刷新、关闭、重启设备数据均不丢失。
2. 存储内容：所有大场对局信息、单局计分明细、时间、时长、统计数据、对局状态。
3. 数据权限：进行中对局可正常记分、提交；已封存对局数据只读，不可编辑、追加单局。
4. 扩展能力：数据结构标准化，预留接口，后续可支持将本地数据提交至服务端。

### 2.10 对局操作规则

#### 2.10.1 新建大场对局

点击【新比赛】按钮，生成全新大场记录，重置全局统计数据、局号为第1局，初始化第一局开始时间，清空历史记录与待提交区域，进入全新对战。

#### 2.10.2 结束并封存整场比赛

1. 操作时机：单局提交完成后可点击【结束本场比赛】。
2. 二次确认：点击按钮弹出确认弹窗，提示文案：`确定要结束本场比赛并封存记录吗？封存后将无法继续新增对局、无法修改数据`，配备「取消」「确定」按钮。
3. 确认操作：点击确定，本场对局标记为「已封存」，所有数据锁定只读；点击取消，关闭弹窗，对局继续进行。

## 三、页面整体布局

页面采用手机竖屏设计，100%适配移动端宽度，顶部全局战绩总览固定悬浮不滚动，其余区域支持上下滚动，控件尺寸适配手指触控。整体自上而下分为六大模块，顶部标题区增设功能按钮。

### 模块一：页面标题与全局操作区

1. 居中展示标题：台球追分记分器。
2. 标题右侧增设两个功能按钮：【新比赛】、【结束本场比赛】。

### 模块二：全局战绩总览（固定悬浮）

1. 展示形式：表格样式，固定两行，分别对应选手1、选手2，数据全部自动计算，只读不可手动编辑。
2. 固定表头顺序：选手 | 犯规 | 分球 | 普胜 | 小金 | 大金 | 额外分 | 总分 | 净分。
3. 字段说明：
  - 选手：展示选手名称，使用专属配色。
  - 犯规：普通犯规+开球犯规累计总次数。
  - 分球/普胜/小金/大金：对应项目累计选择次数。
  - 额外分：让杆加倍产生的分数差值总和。
  - 总分：选手所有得分、扣分合计值。
  - 净分：两名选手总分差值。

### 模块三：本局选手记分操作区

1. 模块标题：本局选手记分选择。
2. 布局：纵向排列两个独立选手操作栏，互不干扰。
3. 单栏结构：
  1. 左侧：带专属配色的选手名称。
  2. 名称右侧：让杆复选框 + 「让杆」文字。
  3. 复选框下方：动态切换计分按钮组。
    - 未勾选让杆：展示 犯规、开球犯规、分球、普胜、小金、大金。
    - 已勾选让杆：展示 分球、普胜、小金、大金（无犯规类按钮）。

### 模块四：本局得分待提交区

1. 模块标题：本局待提交得分（点击标签可单独删除）。
2. 样式：圆角容器，标签横向排列、自动换行。
3. 标签展示：
  - 常规标签：选手名+项目（例：选手1 犯规）。
  - 让杆标签：选手名+让杆+项目（例：选手2 让杆小金）。
4. 交互：单击标签可即时删除，删除后同步解除对应选择限制。
5. 空状态：显示提示文字「暂无得分」。

### 模块五：提交按钮与校验提示区

1. 提示位：位于按钮上方，常态隐藏，校验失败时展示浅红色错误提示，不遮挡操作。
2. 全屏宽按钮：【提交本局成绩】。
  - 待提交区无数据：按钮置灰，点击无效。
  - 校验失败：展示错误提示，不执行提交。
  - 校验通过：正常提交本局数据。

### 模块六：逐局得分历史记录区

1. 模块标题：逐局得分历史记录。
2. 排序与样式：按局号正序排列，局与局之间加分隔线，支持滚动查看。
3. 单局展示内容：局号、本局开始时间、本局结束时间、对局时长、所有计分标签（区分犯规类型、常规/让杆）、两名选手本局基础得分、本局额外分、本局单局总分。
4. 状态区分：已封存对局的历史记录整体置灰，标注「已结束」，无任何编辑操作入口。

## 四、详细交互与流程逻辑

### 4.1 让杆状态切换

勾选/取消「让杆」复选框，实时切换下方计分按钮组；已添加至待提交区的标签不受影响，仅管控后续新增标签类型。

### 4.2 标签添加与删除

1. 添加：点击计分按钮，先执行前置规则校验，合规标签加入待提交区，违规操作直接拦截并短时提示。
2. 删除：点击待提交区内标签即可移除，同时解除该标签对应的次数、互斥限制，可重新选择。

### 4.3 本局提交流程

1. 点击【提交本局成绩】，系统执行全量校验。
2. 校验失败：展示错误提示，保留现有标签，流程终止。
3. 校验通过：
  1. 记录本局结束时间，自动计算对局时长。
  2. 更新顶部全局战绩总览数据。
  3. 生成本局完整明细（含时间、时长、计分标签）。
  4. 将本局数据同步写入IndexedDB本地数据库。
  5. 清空待提交区域，保留让杆复选框勾选状态。
  6. 局号自动+1，将当前局结束时间设为下一局开始时间。
4. 提交完成后，本局数据永久锁定，不可修改、删除。

### 4.4 新建比赛流程

1. 点击【新比赛】按钮。
2. 新建大场对局记录并存入IndexedDB。
3. 清空全场统计数据、历史记录、待提交区，局号重置为第1局。
4. 初始化第一局开始时间，进入全新对局。

### 4.5 结束本场比赛流程

1. 点击【结束本场比赛】，弹出二次确认弹窗。
2. 点击「取消」：关闭弹窗，返回正常记分状态，对局继续。
3. 点击「确定」：对局标记为「已封存」并更新至本地数据库，全场数据锁定只读。

### 4.6 页面初始化逻辑

页面加载时优先读取IndexedDB数据，自动恢复最近一场「进行中」对局的所有状态；若无进行中对局，则自动新建第一场比赛。

## 五、UI样式规范

1. 页面整体风格简洁，无多余装饰、广告与弹窗，所有控件适配移动端触控操作。
2. 选手配色、标签样式、交互规则全程统一。
3. 时间、时长文字使用常规灰色展示；校验错误提示使用浅红色。
4. 已封存对局记录整体置灰，明确区分对局状态。
5. 顶部战绩总览固定悬浮，其余区域正常滚动。

## 六、边界规则与使用限制

1. 固定双人对战，不支持切换对战人数、对战模式。
2. 普通犯规无选择次数限制；开球犯规、分球单局单选手限选1次；普胜、小金、大金三者互斥。
3. 总览页合并两类犯规统计，仅历史明细区分犯规、开球犯规。
4. 仅「进行中」对局可使用记分、提交功能；「已封存」对局仅支持查看，无法新增单局、修改数据。
5. 单局时间、时长由系统自动生成，禁止手动修改。
6. IndexedDB为本地存储，手动清空浏览器数据会删除所有对局记录。
7. 所有已提交的单局数据、已封存的整场对局数据，均不可编辑、删除。

## 七、整体操作流程

1. 打开页面，系统自动读取本地数据，恢复上一场进行中的对局，或自动新建比赛。
2. 如需开启全新对局，点击顶部【新比赛】。
3. 根据对局情况勾选选手「让杆」选项，点击对应计分按钮选择计分项目；误选可点击待提交区标签删除。
4. 确认计分内容无误后，点击【提交本局成绩】，系统自动记录时间、时长并保存至本地。
5. 重复操作完成多局记分，系统自动承接上一局结束时间作为下一局开始时间。
6. 整场对战结束后，点击【结束本场比赛】，在二次确认弹窗中选择「确定」封存本场记录，选择「取消」则继续对战。
7. 对局封存后，可点击【新比赛】开启下一场对局。

## 八、后续扩展方向

1. 新增历史对局列表页，支持查看所有进行中、已封存对局及完整明细。
2. 实现本地对局数据批量/单条上传至服务端，完成云端数据同步。
3. 支持自定义选手名称、对局备注、标签备注功能。
4. 新增数据统计报表，展示全场胜率、平均对局时长、犯规统计等衍生数据。
5. 增加数据导出功能，支持对局记录离线导出。
